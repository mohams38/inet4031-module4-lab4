Colossus Airlines Reservation System

Problem Statement
This project is a seating reservation system for Colossus Airlines. The plane has 24 seats and does two flights each day, one outbound and one inbound. The program lets the user manage seat assignments using menus.

Describe the Solution
I used an array of structures to store the seat information for each flight. Each seat stores the seat number, whether the seat is taken or not, and the passenger’s first and last name. The program starts with a main menu where the user can choose outbound flight, inbound flight, or quit. After that, there is another menu that lets the user check empty seats, see assigned seats in alphabetical order, assign a seat, delete a seat assignment, or go back to the main menu.


Pros
- It is simple and easy to follow
- It works for both outbound and inbound flights

### Cons
- The data only stays while the program is running
- It does not save to a file in this version
